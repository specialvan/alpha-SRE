# Codex Run Card

## Responsibility

Use this card after startup to keep the active task aligned with the package mission and execution rules.

This card is not a replacement for the official startup order. It is the operational reminder once the repository context is already loaded.

## Mission

Build the standalone narrative SRE control plane that can later integrate with `alpha-autopilot`.

## Immediate priorities

1. Follow the official startup order in `CODEX_DEVELOPMENT_GOVERNANCE.md`.
2. Keep governance documents authoritative.
3. Define the replay and consistency model before implementation.
4. Make all changes reversible and reviewable.
5. Keep integration plans separate from current baseline control.

## Working rules

- One task at a time.
- Classify the task before coding.
- Write tests for any new behavior.
- Update the package index when documents move.
- Do not blur baseline and experimental work.

## Exit criteria

- The task is documented.
- The task is tested.
- The task is reversible.
- The task is placed in the correct layer.
