# Archive Structure

## Archive purpose

The archive stores obsolete, superseded, or rejected materials so the active governance path stays clean.

## Archive contents

- retired specifications
- superseded review notes
- deprecated metric proposals
- abandoned experimental designs
- old integration plans

## Rules

- Do not use archived documents as current source of truth.
- If a document moves to archive, record the replacement document in `final_document_index.md`.
- Archived items must not be referenced as baseline requirements.
